## Detailed Description

Team-X is a desktop application (Electron + React) that provides an AI-powered workforce. Users hire AI employees with curated roles (57 available), assign work through tickets, and watch as agents collaborate autonomously while maintaining oversight through Mission Control Dashboard.

**Key Differentiators:**
- **Curated Employee Roles:** Pre-configured AI agents with specialized skills, personalities, and work styles
- **Autonomous Collaboration:** Agents work together, participate in ticket threads, and coordinate without human intervention
- **Budget Governance:** Multi-layer budget controls (monthly caps, per-ticket limits, approval workflows)
- **Provider Flexibility:** Switch between Anthropic (Claude), OpenAI (GPT), and Ollama (local) with automatic failover
- **Multi-Workspace:** Manage multiple companies/clients with data isolation and shared policies
- **Complete Oversight:** Mission Control Dashboard shows all activity, Copilot provides proactive insights

**Target Users:**
- Startups and small businesses
- Agencies managing multiple clients
- Product teams
- Freelancers scaling their capacity
- Developers wanting AI assistance

## Architecture Overview

**Technology Stack:**
- **Desktop Shell:** Electron (Chromium)
- **Frontend:** React with TypeScript
- **State Management:** Zustand
- **Styling:** Tailwind CSS
- **Charts:** Recharts
- **Icons:** Lucide React
- **AI Providers:** Anthropic Claude, OpenAI GPT, Ollama (local LLaMA, Mistral)
- **Protocol:** MCP (Model Context Protocol) for extensions

**Architecture Pattern:**
- MVVM for separation of concerns
- Event-driven architecture for real-time updates
- Plugin system for extensibility
- REST API for automation
- Webhook system for integrations

## Comprehensive Feature Documentation

### 1. Mission Control Dashboard

**Location:** Main view (Ctrl+D / Cmd+D)

**Panels:**
- **Active Runs:** Shows agents currently working, with progress bars and elapsed time
- **Recent Tickets:** Latest tickets with status, assignee, and duration
- **Copilot Insights:** Proactive recommendations categorized as Critical, Warning, Info
- **Budget Status:** Real-time spend vs budget with projections

**Purpose:** Single pane of glass for operational oversight. Operators can monitor all workspace activity from one screen.

**Key Interactions:**
- Click agent run to view stream and artifacts
- Click ticket to open detail view
- Click insight to expand details or dismiss

### 2. Command Palette

**Access:** Ctrl+K (Windows/Linux) or Cmd+K (macOS)

**Capabilities:**
- Natural language command interpretation
- Fuzzy matching across all commands
- Command history (up/down arrows)
- Autocomplete for common actions

**Command Categories:**
- Ticket operations (create, assign, close, etc.)
- Employee operations (hire, fire, list, etc.)
- Budget operations (show spend, set alerts, etc.)
- System operations (settings, integrations, etc.)

**Example Commands:**

### 4. Tickets & Work Management

**Ticket Structure:**
- **Title:** Brief description
- **Description:** Detailed requirements, context, acceptance criteria
- **Assignee:** Primary employee responsible
- **Participants:** Additional employees collaborating (participant wake semantics)
- **Priority:** Low, Normal, High, Critical
- **Dependencies:** Other tickets that must complete first
- **Status:** Open, In Progress, Done, Cancelled

**Ticket Lifecycle:**
1. **Created:** Ticket initialized, assignee notified
2. **In Progress:** Agent run started, employee working
3. **Done:** Work completed, artifacts delivered
4. **Cancelled:** Work discontinued, artifacts preserved

**Task Planner:**
- AI-powered decomposition of large projects
- Input: High-level project description
- Output: Structured ticket hierarchy with dependencies
- Example: "Build a React dashboard" → 12 tickets with dependencies

### 5. Agent Runs

**Agent Run Process:**
1. **Initialization:** Provider selected, runtime configured
2. **Execution:** Agent reads ticket, uses tools to complete work
3. **Tool Use:** Read files, write files, run commands, search code
4. **Collaboration:** Agent may ask questions, request clarification
5. **Completion:** Work delivered, artifacts created, ticket marked done

**Agent Stream:**
Real-time feed showing:
- Agent's thoughts and reasoning
- Tool calls (Read, Write, Bash, etc.)
- Progress updates and milestones
- Errors and recoveries
- Final summary and artifacts

**Artifacts:**
Deliverables created during agent run:
- Code files
- Documentation
- Test results
- Reports
- Screenshots (for design work)

**Approval Workflows:**
- Write operations require operator approval
- Budget overrides require confirmation
- Ticket creation from routines can be batch-approved

### 6. Autonomy Control Plane

**Components:**

**Runtimes:** Execution environments for agents
- local-default: Standard runtime with file/bash/search tools
- bash-runtime: Custom shell execution runtime
- node-runtime: Node.js code execution
- python-runtime: Python code execution

**Routines:** Automated recurring tasks
- Schedule: Cron expression (e.g., "0 9 * * MON" = Monday 9am)
- Work template: Ticket template for routine-generated work
- Budget cap: Monthly spend limit per routine
- Examples: Daily code review, Nightly data sync, Weekly summary, Security scan

**Budgets:** Multi-layer spend controls
- Monthly budget: Workspace cap
- Per-ticket budget: Individual ticket limit
- Routine budget: Per-routine monthly cap
- Approval workflow: Override requests for approval

**Approvals:** Governance for sensitive operations
- Budget overrides
- Write operations (file creation/modification)
- Routine-generated tickets
- Provider failover triggers

**MCP Servers:** Model Context Protocol integrations
- Extend agent capabilities with custom tools
- Examples: Database queries, web search, API integrations
- Management: Add/remove servers, configure connections

**Doctor:** Health diagnostics
- Database integrity check
- Runtime posture assessment
- Provider health verification
- MCP server status
- Budget blocker detection
- Recovery readiness validation

## Troubleshooting Guide

### Common Issues

**Agent stuck running:**
- Check provider status
- Review agent run log for errors
- Cancel run if needed
- Use improvement loop to prevent recurrence

**Budget exhausted:**
- Approve override if legitimate
- Review spend breakdown
- Adjust budgets for future
- Use cheaper providers

**Employee performance issues:**
- Review agent run history
- Provide feedback in ticket thread
- Adjust system prompt

## Glossary (Key Terms)

**Agent:** AI entity powered by LLM that works on tickets autonomously
**Agent Run:** Single execution of an agent working on a ticket
**Artifact:** Output produced by agent run (code, docs, reports)
**Assignee:** Employee primarily responsible for a ticket
**Autonomy Control Plane:** Governance system (runtimes, routines, budgets, approvals)
**Budget:** Spend limit (monthly, per-ticket, per-routine)
**Command Palette:** NLU-driven command interface (Ctrl+K/Cmd+K)
**Copilot:** Proactive intelligence analyzer surfacing insights
**Doctor:** Health diagnostic tool in Autonomy Control Plane
**Employee:** AI agent with curated role (57 available)
**MCP:** Model Context Protocol for extending agent capabilities
**Mission Control:** Real-time operations dashboard
**Participant:** Employee added to ticket for collaboration (not primary assignee)
**Provider:** AI model provider (Anthropic, OpenAI, Ollama)
**Routine:** Automated recurring task executed on schedule
**Runtime:** Execution environment for agent (tools, file access, commands)
**Ticket:** Work unit assigned to employee
**Workspace:** Company container (employees, tickets, budgets, data)
